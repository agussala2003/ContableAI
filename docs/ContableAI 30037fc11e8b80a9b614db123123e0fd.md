# 📄 Documento de Definición de Producto: ContableAI (Versión Final)

## 1. Visión del Producto

**ContableAI** es una plataforma SaaS diseñada para automatizar la **conciliación bancaria** en estudios contables.

Su objetivo es eliminar el punteo manual línea por línea, utilizando Inteligencia Artificial para interpretar el extracto bancario y cruzar información fiscal **solo cuando es necesario (Impuestos)**, asegurando que el saldo contable coincida siempre con el saldo real del banco.

---

## 2. El Problema que Resolvemos

- **Pérdida de Tiempo:** Los contadores pierden horas clasificando gastos recurrentes que cambian de descripción (ej: "MERCADOPAGO*COTO" vs "COTO C.I.C.S.A.").
- **Ambigüedad Fiscal:** En el extracto bancario, todos los impuestos dicen "DEBIN AFIP" o "VEP". Es difícil saber si ese pago corresponde a IVA, Ganancias o Cargas Sociales sin entrar a la página de AFIP a investigar.
- **Gestión de la Realidad:** Los sistemas actuales se traban si un gasto no tiene factura asociada. ContableAI prioriza la realidad financiera: si la plata salió, se registra.

---

## 3. Funcionalidades Principales (El Alcance)

### A. El Motor de Clasificación Bancaria (El Core)

Procesa el 100% de los movimientos del extracto.

- **Input Universal:** Acepta Excel, CSV y PDF de bancos y billeteras virtuales.
- **Lógica de "Realidad Financiera":**
    - Si el gasto es claro (ej: "Librería"), se imputa a la cuenta de resultado correspondiente.
    - Si es un gasto "sin respaldo" o retiro de dinero (ej: Transferencia a un particular sin factura), la IA lo categoriza automáticamente en cuentas de gestión interna (ej: **"Cuenta Particular Socio"**, **"Adelantos"** o **"Gastos a Rendir"**).
    - **Objetivo:** Que el saldo del banco cierre perfecto al centavo, sin importar si hay factura o no.

### B. Módulo de Identificación de Impuestos (Integración AFIP)

Este módulo se activa **exclusivamente** cuando el sistema detecta un pago a organismos fiscales (AFIP, ARBA, AGIP).

- **El Problema:** El banco dice "DEBIN AFIP - $120.000".
- **La Solución:** El sistema consulta los datos de AFIP (Declaraciones Juradas y Pagos) para decirnos: *"Este débito corresponde al F.931 (Sueldos) de Enero 2026"*.
- **Modos de Uso:**
    1. **Manual (MVP):** El usuario sube el Excel de "Mis Presentaciones" y el sistema hace el cruce.
    2. **Automático:** Conexión directa vía Certificado Digital para leer las DDJJ sin intervención.

### C. Memoria Inteligente

- El sistema aprende de las correcciones. Si el contador define que "Transferencia a Juan Pérez" es siempre "Alquiler", el sistema no vuelve a preguntar.

### D. Exportación Contable

- Generación de archivos compatibles con sistemas de gestión (Holistor, Bejerman, etc.) para importar los asientos ya procesados.

---

## 4. Flujo de Uso (La Experiencia del Usuario)

1. **Carga:** El contador sube el extracto bancario.
2. **Procesamiento Inteligente:**
    - **¿Es un pago a AFIP?** -> El sistema busca el respaldo en los datos de impuestos para identificar qué se pagó.
    - **¿Es otro gasto?** -> La IA deduce la categoría por el nombre (ej: "Shell" -> "Combustibles").
    - **¿Es un gasto sin clasificar/negro?** -> La IA lo asigna a "Cuenta Socio" o "Varios" para que el banco cierre.
3. **Revisión (Semáforo):**
    - 🟢 **Verde:** Impuestos identificados y gastos recurrentes confirmados.
    - 🟡 **Amarillo:** Gastos nuevos o dudosos que requieren el "Ojo del Contador".
    - 🔴 **Rojo:** Movimientos complejos que requieren asignación manual.
4. **Cierre:** Exportación del lote de asientos.

## 5. Arquitectura Técnica Resumida

- **Frontend:** Angular + Tailwind CSS.
- **Backend:** .NET 8 (C#) con Clean Architecture.
- **IA:** Microsoft Semantic Kernel (para la lógica de clasificación).
- **Datos:** SQL Server + Qdrant (Vector Database).
- **Integración:** Librerías de OCR para PDFs y Parsing para Excel/CSV.
